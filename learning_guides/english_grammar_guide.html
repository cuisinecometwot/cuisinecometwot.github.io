📅 English Tenses

Present Tenses

Present Simple

Subject + Base Verb (+ s/es for 3rd person singular)
Technical Examples:

General Truths: "The CPU processes data sequentially."

Habits/Routines: "I usually commit code before pushing to the repository."

Scheduled Events: "The server maintenance starts at 2 AM every Sunday."

// Database connection example const connection = mysql.createConnection({ host: 'localhost', user: 'root', password: 'password' }); // Present Simple usage in comments // This function always returns a promise function fetchData() { return new Promise((resolve, reject) => { // Implementation here }); }

Present Continuous

Subject + am/is/are + Verb-ing
Technical Examples:

Actions in Progress: "I am currently debugging the authentication module."

Temporary Actions: "The system is running a backup process right now."

Future Arrangements: "We are deploying the new version next week."

// Present Continuous in technical context class SystemMonitor { constructor() { this.isRunning = false; this.currentProcess = null; } startMonitoring() { if (this.isRunning) { console.log("Monitoring is already running"); return; } this.isRunning = true; console.log("System monitoring is starting..."); // Start monitoring process this.currentProcess = setInterval(() => { this.checkSystemHealth(); }, 5000); } }

Present Perfect

Subject + have/has + Past Participle
Technical Examples:

Completed Actions with Present Relevance: "I have implemented the new API endpoint."

Experience: "She has worked with React for three years."

Unfinished Time Periods: "We have deployed five updates this month."

// Present Perfect in technical documentation class UserService { constructor() { this.users = []; this.totalUsers = 0; } addUser(user) { // Check if user already exists if (this.users.find(u => u.email === user.email)) { throw new Error("User already exists"); } this.users.push(user); this.totalUsers++; // Log the action console.log(`User ${user.email} has been added successfully`); console.log(`Total users: ${this.totalUsers}`); } }

Past Tenses

Past Simple

Subject + Past Tense Verb
Technical Examples:

Completed Actions: "I deployed the application yesterday."

Sequential Actions: "First, I cloned the repository, then I created a new branch."

Past Habits: "We used to deploy manually before implementing CI/CD."

// Past Simple in error handling async function processData() { try { // Attempt to fetch data const response = await fetch('/api/data'); if (!response.ok) { throw new Error(`HTTP error! status: ${response.status}`); } const data = await response.json(); return data; } catch (error) { // Log what happened console.log(`Error occurred: ${error.message}`); console.log(`Timestamp: ${new Date().toISOString()}`); // Return fallback data return getDefaultData(); } }

Future Tenses

Will Future

Subject + will + Base Verb
Technical Examples:

Predictions: "The new algorithm will improve performance by 30%."

Spontaneous Decisions: "I'll help you debug that issue."

Promises/Offers: "I will review your pull request by tomorrow."

// Will Future in promise handling class TaskScheduler { constructor() { this.tasks = []; this.isRunning = false; } addTask(task) { this.tasks.push(task); console.log(`Task "${task.name}" will be executed at ${task.scheduledTime}`); } async executeTasks() { if (this.isRunning) { console.log("Task execution is already in progress"); return; } this.isRunning = true; for (const task of this.tasks) { try { console.log(`Executing task: ${task.name}`); await task.execute(); console.log(`Task "${task.name}" will be marked as completed`); } catch (error) { console.error(`Task "${task.name}" will be retried due to error: ${error.message}`); } } this.isRunning = false; } }

🏗️ Grammar Structures

Conditional Sentences

Zero Conditional

If + Present Simple, Present Simple
Technical Examples:

"If you restart the service, it clears the cache."

"If the database connection fails, the application logs an error."

// Zero Conditional in system behavior class CacheManager { constructor() { this.cache = new Map(); this.maxSize = 1000; } set(key, value) { // If cache is full, remove oldest entry if (this.cache.size >= this.maxSize) { const firstKey = this.cache.keys().next().value; this.cache.delete(firstKey); } this.cache.set(key, value); } get(key) { // If key exists, return value; if not, return null if (this.cache.has(key)) { return this.cache.get(key); } return null; } }

First Conditional

If + Present Simple, will + Base Verb
Technical Examples:

"If the test passes, I will merge the branch."

"If the server responds, the client will process the data."

// First Conditional in error handling class APIClient { constructor(baseURL) { this.baseURL = baseURL; this.retryCount = 0; this.maxRetries = 3; } async makeRequest(endpoint) { try { const response = await fetch(`${this.baseURL}${endpoint}`); if (response.ok) { this.retryCount = 0; // Reset retry count on success return await response.json(); } else { throw new Error(`HTTP ${response.status}: ${response.statusText}`); } } catch (error) { if (this.retryCount < this.maxRetries) { this.retryCount++; console.log(`Request failed, will retry (${this.retryCount}/${this.maxRetries})`); return this.makeRequest(endpoint); // Retry } else { throw new Error(`Max retries exceeded: ${error.message}`); } } } }

Passive Voice

Present Passive

Subject + am/is/are + Past Participle
Technical Examples:

"The data is processed by the algorithm."

"New features are developed by the team."

// Present Passive in system architecture class DataProcessor { constructor() { this.processors = []; } addProcessor(processor) { this.processors.push(processor); } processData(data) { let processedData = data; // Data is processed by each processor in sequence for (const processor of this.processors) { processedData = processor.process(processedData); } return processedData; } }

Modal Verbs

Can/Could (Ability/Possibility)

Subject + can/could + Base Verb
Technical Examples:

"The system can handle 1000 concurrent users."

"This algorithm could be optimized further."

// Can/Could in system capabilities class SystemCapabilities { constructor() { this.maxConcurrentUsers = 1000; this.currentUsers = 0; } canAcceptNewUser() { return this.currentUsers < this.maxConcurrentUsers; } acceptUser() { if (this.canAcceptNewUser()) { this.currentUsers++; return true; } return false; } optimizePerformance() { // The system could be optimized by implementing caching if (this.currentUsers > this.maxConcurrentUsers * 0.8) { console.log("System load is high, performance could be optimized"); this.enableCaching(); } } enableCaching() { console.log("Caching enabled to improve performance"); } }

/** * User API Endpoints * * This API provides user management functionality. * All endpoints require authentication via JWT token. */ // GET /api/users // This endpoint returns a list of all users // The response includes pagination metadata app.get('/api/users', authenticateToken, async (req, res) => { try { const page = parseInt(req.query.page) || 1; const limit = parseInt(req.query.limit) || 10; const users = await User.find() .limit(limit) .skip((page - 1) * limit); const total = await User.countDocuments(); res.json({ users: users, pagination: { page: page, limit: limit, total: total, pages: Math.ceil(total / limit) } }); } catch (error) { res.status(500).json({ error: "Internal server error" }); } });

Error Messages

Present Simple for Error Descriptions

Error Message Examples:

"The file does not exist in the specified directory."

"Authentication fails when the token expires."

"The database connection times out after 30 seconds."

// Error handling with clear grammar class ErrorHandler { constructor() { this.errorMessages = { 'FILE_NOT_FOUND': 'The requested file does not exist in the system.', 'AUTHENTICATION_FAILED': 'Authentication fails when credentials are invalid.', 'DATABASE_TIMEOUT': 'The database connection times out after 30 seconds.', 'INVALID_INPUT': 'The input data does not match the expected format.', 'PERMISSION_DENIED': 'Access is denied due to insufficient permissions.' }; } handleError(errorCode, context = {}) { const baseMessage = this.errorMessages[errorCode]; if (!baseMessage) { return 'An unexpected error occurs during operation.'; } // Add context-specific information let detailedMessage = baseMessage; if (context.filePath) { detailedMessage += ` File path: ${context.filePath}`; } if (context.userId) { detailedMessage += ` User ID: ${context.userId}`; } return detailedMessage; } }

✍️ Practice Exercises

Exercise 1: Fill in the Correct Tense

Complete the sentences with the correct verb form:

1. The server _____ (run) continuously since last Monday.

2. I _____ (work) on this bug when the system crashed.

3. The new feature _____ (be) deployed next week.

4. If the test passes, I _____ (merge) the code.

5. The API _____ (return) JSON data by default.

Answers:
1. has been running (Present Perfect Continuous)
2. was working (Past Continuous)
3. will be (Future Simple Passive)
4. will merge (First Conditional)
5. returns (Present Simple)

Exercise 2: Convert to Passive Voice

Rewrite these sentences in passive voice:

1. The developer writes the code.

2. The system processes the data.

3. The team will deploy the application.

4. The algorithm has optimized the performance.

5. The user must provide valid credentials.

Answers:
1. The code is written by the developer.
2. The data is processed by the system.
3. The application will be deployed by the team.
4. The performance has been optimized by the algorithm.
5. Valid credentials must be provided by the user.
