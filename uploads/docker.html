Docker

CommandDescription
docker imagesLists all local Docker images.
docker image inspect [image_name]Provides detailed information about a specific image.
docker pull [image_name]Downloads an image from Docker Hub.
docker rmi [image_name]Deletes an image from your local system (use with caution).
docker search [term]Searches for images on Docker Hub.
docker run [options] [image_name]Creates and starts a container from an image. (e.g., docker run -it ubuntu bash for an interactive Ubuntu container)
docker psShows all running containers.
docker ps -aLists all containers, including stopped ones.
docker stop [container_id]Stops a running container.
docker start [container_id]Starts a stopped container.
docker rm [container_id]Deletes a container (use with caution, data is lost).
docker inspect [container_id]Provides detailed information about a specific container.
docker attach [container_id]Attaches your terminal to a running container.
docker exec -it [container_id] bashRuns a command inside a running container (interactive).
docker build -t [image_name] .Builds an image from a Dockerfile in the current directory.
docker network lsShows all Docker networks.
docker network connect [network_name] [container_id]Connects a container to a specific network.
docker volume create [volume_name]Creates a named volume for persistent data storage.
docker volume lsShows all Docker volumes.
docker volume inspect [volume_name]Provides detailed information about a specific volume.
docker run -v [volume_name]:[container_path] [image_name]Mounts a volume to a specific path inside a container.
docker help [command]Detailed information on a specific command.
docker-compose up -dBuilds, (re)creates, starts and attaches to containers for all services in docker-compose.yml file. Option -d for detached mode (run in the background).
docker-compose downStops all running containers created by docker-compose up and removes networks used by them.
docker-compose psLists all services defined in your docker-compose.yml file with their current states (running, stopped, etc.).
docker-compose logsShows logs from all the containers managed by docker-compose in real time. You can specify a service name to see logs only for that service.
docker-compose buildBuilds images for all services defined in your docker-compose.yml file.
docker-compose run [service_name]Runs a container for a specific service in detached mode. Useful for running one-off commands or processes within a service container.
docker-compose exec [service_name] [command]Executes a command within a running service container.
